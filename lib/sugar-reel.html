🎬 SugarReel

Terminal video player

prior art: tplay · glyph · video-to-ascii video ascii-art half-block sixel ffmpeg

Plays a video file — mp4 and more — by decoding frames on the fly and rendering each one to the terminal as ASCII, ANSI 256-color, truecolor half-blocks, sixel, or kitty graphics. Like mpv -vo tct, but PHP, and reusing the SugarCraft rendering stack throughout.

Install

composer require sugarcraft/sugar-reel

Quickstart

use SugarCraft\Reel\Player;

$player = Player::open('clip.mp4', cols: 80, rows: 24);
(new \SugarCraft\Core\Program($player))->run();

Design

ffmpeg decode pipeShells to the ffmpeg binary via proc_open for raw rgb24 frames, pre-scaled and fps-decimated by ffmpeg so PHP never decodes a codec itself.
Pure-PHP GIF fallbackWhen ffmpeg is absent, falls back to candy-flip's GD-based decoder for .gif sources.
Reused renderersHalf-block / sixel / kitty / iTerm2 rendering delegates to candy-mosaic rather than being re-implemented.
Color mappingcandy-palette maps RGB to 256-color and truecolor SGR for non-truecolor TTYs.
TEA runtimePlayback is a candy-core Model paced by Cmd::tick() — no busy-waiting.
Audio syncAudio subprocess (ffplay / mpv --no-video) is the master wall clock; video frame-skips to track it.
