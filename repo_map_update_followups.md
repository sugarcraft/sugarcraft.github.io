## Overview

This document audits every SugarCraft lib **not** migrated in steps 9-29 (those are the "already-done" libs). It maps each remaining lib to the 8 shared packages it could adopt, with rationale and effort estimates.

The 8 shared packages:
- **candy-ansi** — byte-loop ANSI parsing (Parser, OscHandler, CsiHandler, SgrState)
- **candy-buffer** — cell-grid rendering & string composition (Buffer, Cell, DiffOp, Style, Hyperlink)
- **candy-layout** — constraint-based sizing (LayoutSolver, Region, Direction, Constraint*)
- **candy-mouse** — mouse hit-testing & decode (MouseMsg, Zone, Manager)
- **candy-input** — input decode & KeyEvent (InputDriver, KeyEvent, SymbolicKey)
- **candy-fuzzy** — filter/search via Smith-Waterman (Matcher, MatchResult)
- **candy-async** — ReactPHP helpers (AsyncOps, CancellationSource, CancellationToken)
- **candy-testing** — golden-file snapshot tests (assertGoldenAnsi, GoldenRenderTest)

## Per-lib opportunities

### candy-pty

| Package | Benefit | Effort |
|---------|---------|--------|
| candy-async | PtyPool/MultiPump use manual Deferred/Timer loops — AsyncOps::withTimeout() + CancellationToken would simplify cancellation | M |
| candy-input | Pty read/write dispatches raw bytes — StreamInputDriver/KeyEvent decode would standardize input path | M |

**Notes:** candy-pty is the most foundational async PTY lib. It predates candy-async and manually wires Loop + Deferred for timeout/cancellation. Adopting candy-async would reduce boilerplate in MultiPump and PtyPool. Input decoding is currently raw byte-oriented; candy-input would provide symbolic key decode.
